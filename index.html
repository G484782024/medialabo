本テーマについて

先行する情報工学の専門科目の基礎知識を応用して実践的な能力を習得することが目的である。このHTML等を用いたWebデザイン作成では、課題に対する経血能力の育成を目標としている。

プロジェクト課題について

世界(12ヶ所)の天気がわかるサイト。検索方法はテキストボックスではなく、都市名（国名）のプルダウンメニューによって検索することができます。また、天候、最高気温、湿度といった重要な情報が一目でわかるよう、色分けを行いました。このサイトの最も重要な情報である、国名は文字だけの出力ではなく、GIF画像での出力をすることにより、一目でわかりやすくするだけでなく、動的な、シンプルすぎないUIとした。

各授業回の学習事項

第1回 WWW と HTML

HTMLは、要素によって構成されており、その中でのブロック要素、インライン要素に分類される。HTMLではコード上に入力しても、Webページ上では反映されない特殊文字が存在するため、<, >, ", ', &などは別の記述を使用する必要がある。

演習の成果(1): 演習1-3: HTML文書の作成
演習の成果(2): 演習1-4: HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

CSSは、 Webサイトのデザインを決める役割を果たしており、カラーや配置など、様々なデザインができる。また、CSSを書く際には、セレクタという対象要素を指定する記述が多くあり、一部または、条件に当てはまるものを変更したい場合は、セレクタを利用する。

演習の成果: 演習2-3: ウェブブラウザ説明ページの装飾

第3回 JavaScript の基本

JavaScriptは、Webページ上で複雑な機能を実現するプログラミング言語であり、目にみえるWebページ上の動作以外にも、コンソールの動きに変化を加えることができる。javaのように、for文やif文を使用できるが、違う点として、コンパイルを行う必要がないという点がある。実行に手間がかからないという利点があるが、デバッグを行うのが大変というデメリットもある。

演習の成果: 演習3-6: オブジェクトの参照

第4回 DOM と JavaScript

JavaScriptでは、前回行ったコンソール上での動作に加えて、Webページそのものを動作させることのできる言語である。その一例として、テキストボックス、ボタンを表示させ、クリックによって動作させることができる。JavaScriptでプログラミングを行う際には、エディタで先に記述するのではなく、Webページのコンソール上で動作を確認してからエディタに記述するのが、ミスの少ないプログラムを書くコツである。

演習の成果: 演習4-10: 足し算ページ

第5回 イベント処理

JavaScriptのDOMの機能の一部として、要素の新規作成、要素の編集、要素の削除が行える。要素の新規作成や削除の機能によって、HTMLファイルの記入を最小限に抑え、JavaScript上でデザインできる。また、要素の削除の機能によって、JavaScript特有のデザインの変更などが行えるようになる。

演習の成果: 演習5-7: DOMプログラミング

第6回 プロジェクト課題を完成させよう

学習内容としては、AJAX通信によるデータの要求、受取を行った。XMLHttpRequestというオブジェクトを使用して行う方法があるが、難易度が高いため、今回はaxiosというライブラリを使用して行った。また、プロジェクト課題では、idの入力による検索では、使い勝手が悪いと感じたため、プルダウンメニューによって、検索キーと国名を紐づけることによって、わかりやすいUIにした。


工夫点，苦心点，強調点，感想

まず、本講義のまとめのサイトとなる、「index.html」については、文字数の多い「本テーマについて」と、比べて文字数の少ない「プロジェクト課題について」「数当てゲーム」を見やすく、綺麗に配置するため、grid-template-columnsを使用し、文字数の多いもののみ画面横幅いっぱい使用するようなデザインとした。また、 全体を含むdivと一部文章を含むclassを別々にborderを設定することにより、1番外枠は黒の実線、その1つ内側はグレーの点線となり、良いデザインとなった。index.htmlに関して苦労した点として、gridを設定しようとした際に、classによって、配置を割り振るプログラム、grid-template-areasを使用しようとしたが、画面サイズ、解像度によって配置が変更されてしまい、同じ環境でないと想定している動作はしないという状況になった点である。その点を解決するために、grid-template-columnとrowsをautoとすることにより、自分の知識外の範囲もカバーすることが出来た。
次に、力を入れたprojectに関してである。まず一番苦労した点として、授業内で行った、「HTMLで書いた内容を、JavaScriptに書き写す」という内容である。授業を聞いているだけでは簡単に聞こえたが、動作させる順番や、id,classの割り振り、要素内の要素を認識させるなど、HTMLだけ使用する際には簡単に出来たことが、JavaScriptに移行するというだけでここまで難易度が上がるのか、と実感した。次に工夫点として、1つは検索方式である。最終回の授業まではtype＝textの検索方式を使用していたが、実際に使用するとなった場合、テキストボックスにidを入力するという手間が面倒だと感じたため、都市名（国名）のプルダウン方式を採用した。このおかげで人目で分かりやすく、楽にできる検索方式となった。2つ目は色分けである。元々背景色を追加していなかったが、その状態だと、「このサイトにおいてどこが重要な要素であるのか」というのが分かりにくいサイトとなっていたため、天気、国名、湿度のところに分かりやすくbackground-colorを設定した。最後に、このサイト上での1番重要な要素の一つである、検索した国名を画面下に表示したという点である。元々は適当な画像の表示としていたが、作成途中に「この画像を国旗のGIF画像にしたら、内容がわかりやすくなるだけでなく、サイトとしても動きが追加されることにより、つまらないサイトを脱却できるのではないか」と思ったことからこの工夫をした。やり方として、javaを勉強した際の知識であるSwitch文を使用し、caseでプルダウンメニューで使用したvalueを呼び出し、動作を行うというものにした。
最後に、感想としては、元々Webデザインには興味があり、やってみたいと思っていた内容だったが、期待していたよりも奥が深く、やれることが増えていくにつれて興味が湧き、楽しくなってきたので、今回の情報メディア実験にてさわれて良かったと思う。まだGitのことや、プログラム内のGridなどわからないことだらけですので、自身で今後もやっていきたいと思いました。そのくらいやっていて楽しかったです。
